# 金融场景 RAG + 微调 项目 — 需求规格说明书

## 1. 引言

### 1.1 编写目的

明确金融场景下「RAG + 领域微调」产品的业务范围、功能与非功能需求，作为产品设计、开发、测试与验收的依据。

### 1.2 项目背景

金融机构内部存在大量非结构化文档（研报、公告、制度、产品说明）。通用大模型存在知识截止、术语偏差与幻觉风险。本项目通过 **RAG** 提供可溯源的知识增强，通过 **微调** 提升金融任务理解与表达一致性。

### 1.3 术语

| 术语 | 说明 |
| --- | --- |
| RAG | Retrieval-Augmented Generation，检索增强生成 |
| SFT | Supervised Fine-Tuning，监督微调 |
| LoRA / QLoRA | 低秩适配 / 量化低秩适配 |
| Chunk | 经分块后的文本单元及元数据 |

### 1.4 参考范围

本需求覆盖 **MVP（最小可行产品）** 与 **扩展能力** 两级；扩展项在文中标注为「二期」。

## 2. 干系人与角色

| 角色 | 诉求摘要 |
| --- | --- |
| 业务用户 | 自然语言问答、摘要、条款对比，答案可信、有出处 |
| 知识管理员 | 文档上架、更新、下线、质量与分类管理 |
| 算法工程师 | 数据管线、索引、训练与评测可复现 |
| 安全合规 | 审计、脱敏、权限、留痕与数据出境策略 |
| 运维 | 可观测、容量与版本回滚 |

---

## 3. 总体目标与成功标准

### 3.1 业务目标

- 在授权语料范围内，用户问题 **首答命中率** 与 **引用正确率** 达到约定阈值（见验收指标）。
- 支持典型金融任务：事实问答、多文档摘要、条款定位、风险点列举（模板化输出）。
- 微调后模型在 **金融术语理解** 与 **指定输出格式** 上优于仅 RAG + 通用基座的对照组。

### 3.2 成功标准（验收方向）

| 维度 | MVP 建议指标（可按资源调整） |
| --- | --- |
| 检索 | Top-5 内相关片段召回率 ≥ 约定值（自建标注集） |
| 生成 | 带引用的问题中，引用与陈述一致率 ≥ 约定值 |
| 延迟 | P95 端到端延迟 ≤ 约定秒数（并发与硬件写入 SLA） |
| 安全 | 越权访问 0 容忍；全链路审计日志可查 |

## 4. 功能需求

### 4.1 用户与权限（FR-AUTH）

| ID | 描述 | 优先级 |
| --- | --- | --- |
| FR-AUTH-01 | 用户登录与租户隔离（多租户可选） | P0 |
| FR-AUTH-02 | 知识库级/文档级访问控制 | P0 |
| FR-AUTH-03 | 管理员角色与普通用户角色分离 | P0 |
| FR-AUTH-04 | 操作审计（谁、何时、对何资源、何动作） | P0 |

### 4.2 文档与知识库管理（FR-KB）

| ID | 描述 | 优先级 |
| --- | --- | --- |
| FR-KB-01 | 支持 PDF、HTML、TXT、Markdown 等格式上传（二期可扩 Office） | P0 |
| FR-KB-02 | 文档解析状态可查询（排队/解析中/成功/失败及原因） | P0 |
| FR-KB-03 | 文档版本与重新索引（更新后触发增量或全量策略可配置） | P1 |
| FR-KB-04 | 元数据：标题、来源、发布日期、文档类型、标签 | P0 |
| FR-KB-05 | 文档下线与索引同步删除 | P0 |

### 4.3 RAG 问答（FR-RAG）

| ID | 描述 | 优先级 |
| --- | --- | --- |
| FR-RAG-01 | 多轮对话，携带短期上下文 | P0 |
| FR-RAG-02 | 流式输出回答 | P0 |
| FR-RAG-03 | 回答附带可点击或可展示的引用（文档名、页码/段落、摘录） | P0 |
| FR-RAG-04 | 检索无相关结果时，明确拒答或建议换问法（非编造） | P0 |
| FR-RAG-05 | 混合检索（向量 + 关键词/BM25）可配置开关 | P1 |
| FR-RAG-06 | 重排（Reranker）可配置开关 | P1 |
| FR-RAG-07 | 查询改写/多查询扩展（二期） | P2 |

### 4.4 微调与模型管理（FR-FT）

| ID | 描述 | 优先级 |
| --- | --- | --- |
| FR-FT-01 | 从审核通过的语料/标注导出标准指令格式（如 ShareGPT / Alpaca JSONL） | P0 |
| FR-FT-02 | 支持 LoRA/QLoRA 训练任务提交、超参模板、GPU 资源标签（可与外部训练平台对接） | P1 |
| FR-FT-03 | 模型版本注册：基座版本、adapter 版本、数据版本、评估摘要 | P1 |
| FR-FT-04 | 推理侧可配置「基座 + 指定 adapter」或仅基座（与 RAG 组合） | P1 |
| FR-FT-05 | 偏好对齐（DPO）数据接入（二期） | P2 |

### 4.5 评测与运营（FR-EVAL）

| ID | 描述 | 优先级 |
| --- | --- | --- |
| FR-EVAL-01 | 离线评测集管理（问题、标准答案、 gold 文档 ID 可选） | P1 |
| FR-EVAL-02 | RAG 指标：召回、上下文精确度等（可用 ragas 等） | P1 |
| FR-EVAL-03 | 线上抽样与人工反馈标注回流（二期） | P2 |

### 4.6 合规与风控输出（FR-RISK）

| ID | 描述 | 优先级 |
| --- | --- | --- |
| FR-RISK-01 | 系统/用户级免责声明与话术策略可配置 | P0 |
| FR-RISK-02 | 敏感意图识别（投资建议、法律意见等）降级为信息说明或拒答（策略可配） | P1 |
| FR-RISK-03 | 输出日志不含明文 PII（或按策略脱敏存储） | P0 |

## 5. 非功能需求

### 5.1 性能与可用性

- API 可用性目标：例如 99.5%（按部署等级定义）。
- 支持水平扩展的无状态 API；长连接流式需配置超时与心跳。

### 5.2 安全

- 传输 HTTPS；静态密钥不入库明文；最小权限原则。
- 上传文件大小与类型白名单；可选杀毒集成。

### 5.3 可维护性

- 配置外置（环境变量/配置中心）；索引与模型版本可回滚。
- 关键链路 Trace ID，便于与 LangSmith 或自建日志关联。

### 5.4 兼容性

- 明确支持的 Python、CUDA、驱动与依赖版本矩阵（见开发文档）。

---

## 6. 约束与假设

- 语料均为 **已获授权** 可索引与可用于训练的数据；客户数据单独租户存储。
- MVP 可先对接 **一种** 向量库与 **一种** 基座模型，降低集成面。
- 微调算力可为离线批量；不要求与在线 API 同集群。

---

## 7. 范围外（本期不做或可选）

- 自动交易执行、实时行情决策（本系统定位为知识辅助）。
- 完全自动化合规裁决（需人工复核流程对接时另立项）。
- 多模态图表深度理解（二期可评估）。

---

## 8. 验收与测试要点

- 功能：权限、索引生命周期、问答引用、拒答、流式。
- 安全：越权用例、审计完整性。
- 回归：索引重建后问答一致性抽样。
